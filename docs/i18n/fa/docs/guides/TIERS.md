# سطوح RouteChi — راهنمای کاربر

RouteChi بیش از ۲۰۷ ارائه‌دهندهٔ پشتیبانی‌شده را در ۳ سطح اقتصادی دسته‌بندی می‌کند. هر درخواست به‌ترتیب از این سطوح عبور می‌کند تا یکی از آن‌ها با موفقیت پاسخ دهد — شما ارزان‌ترین پاسخ ممکن را دریافت می‌کنید بدون آن‌که هرگز کد fallback بنویسید.

## سطح ۱ — اشتراک

**ارائه‌دهنده‌هایی که از قبل برایشان هزینه پرداخت می‌کنید.** RouteChi از تمام سهمیه استفاده می‌کند پیش از آنکه منقضی شود.

| ارائه‌دهنده                        | چرا سطح ۱                                    |
| ---------------------------------- | -------------------------------------------- |
| Claude Code OAuth                  | Anthropic Pro/Team — نرخ ثابت، اغلب استفاده‌نشده |
| OpenAI Codex (اشتراک ChatGPT)      | Plus/Team شامل سهمیهٔ Codex است              |
| GitHub Copilot                     | به ازای هر کاربر — سهمیه ماهانه بازنشانی می‌شود |
| Cursor IDE                         | سهمیهٔ طرح Pro                               |
| Antigravity / Windsurf             | سهمیه‌های داخلی                              |

**استراتژی**: نخست درخواست‌هایی که با نقاط قوت مدل هم‌خوانی دارند به این سطح مسیریابی شوند. ردیاب سهمیه بازنشانی نزدیک را پایش می‌کند؛ استراتژی‌های ترکیبی `reset-aware` و `subscription` بر این اساس اولویت‌بندی می‌کنند.

## سطح ۲ — ارزان

**ارائه‌دهنده‌های پرداخت به ازای توکن زیر ۱ دلار در هر میلیون توکن.** مختص کارهای پربحران یا زمانی که سهمیهٔ سطح ۱ به محدودیت می‌رسد.

| ارائه‌دهنده                | قیمت (ورودی/خروجی)   | نقاط قوت             |
| -------------------------- | -------------------- | -------------------- |
| DeepSeek V4 Pro            | $0.27 / $1.10 در ۱M  | کد، استدلال          |
| GLM-4.5                    | $0.60 / $2.20 در ۱M  | زمینهٔ طولانی         |
| MiniMax M1                 | $0.20 / $1.10 در ۱M  | سرعت                 |
| Qwen Coder                 | $0.30 / $1.20 در ۱M  | کد                   |
| OpenRouter (بهینه‌شده برای قیمت) | متغیر                | بیش از ۱۰۰ مدل، پویا |

**استراتژی**: ترکیب `cost-optimized` ارزان‌ترین مدل بر اساس $/token که فیلتر قابلیت وظیفه (بصری، حالت JSON، ابزارها، حداکثر زمینه) را برآورده کند، انتخاب می‌کند.

## سطح ۳ — رایگان

**ارائه‌دهنده‌های بدون هزینه** — سطوح رایگان، برنامه‌های اعتباری، سهمیه‌های روزانه OAuth.

| ارائه‌دهنده       | سهمیه/اعتبار رایگان                   |
| ----------------- | ------------------------------------- |
| Kiro AI           | سطح رایگان Claude (استفادهٔ منصفانه سخاوتمندانه) |
| OpenCode Free     | بدون احراز هویت، محدودیت نرخ سخاوتمندانه |
| Qoder             | OAuth رایگان                          |
| Google Vertex AI  | اعتبار ۳۰۰ دلاری برای حساب جدید       |
| Amazon Q          | سطح رایگان برای کاربران AWS           |
| Pollinations      | API عمومی باز                         |
| Cloudflare AI     | سطح رایگان Workers AI                 |

**استراتژی**: ترکیب `auto` با سقف بودجه هنگامی که سطح ۱ و ۲ شکست می‌خورند یا هنگامی که `useFreeOnly=true` تنظیم شده باشد، به این سطح مسیریابی می‌کند. ارائه‌دهنده‌های رایگان اغلب محدودیت نرخ ضعیف‌تری دارند — قطع‌کن (circuit breaker) آن‌ها را با backoff بازیابی می‌کند.

## پیکربندی سطوح

داشبورد → **Tiers** → ارائه‌دهنده‌های خود را اختصاص دهید. پیش‌فرض‌ها (از `tierDefaults.json`) معقول هستند؛ هنگامی که اشتراک‌های خاصی برای اولویت‌دهی یا ارائه‌دهنده‌هایی برای حذف دارید، ویرایش کنید.

امتیازدهی ۹ عاملی Auto-Combo نیز سطح را در نظر می‌گیرد. به
[`docs/routing/AUTO-COMBO.md`](../routing/AUTO-COMBO.md) مراجعه کنید.

## تلمتری

داشبورد → **Usage** توکن‌های مصرف‌شده به ازای هر سطح در روز را نمایش می‌دهد. از این برای:

- تأیید اینکه سطح ۱ کاملاً به‌کار گرفته شده (در غیر این‌صورت ارزش اشتراک را هدر می‌دهید)
- شناسایی اینکه کدام مدل‌های سطح ۲ بیشترین انتخاب می‌شوند (به ۱-۲ مورد تقلیل دهید)
- راستی‌آزمایی اینکه سطح ۳ در کارهای تست/کاوش صرفه‌جویی می‌کند

## الگوهای رایج

### بار کاری کاملاً رایگان

```json
{
  "strategy": "auto",
  "config": { "auto": { "weights": { "costInv": 0.5, "tierPriority": 0.3 } } }
}
```

به‌شدت به سمت سطح ۳ می‌رود؛ تنها در صورت در دسترس نبودن سطح ۳ از سطح ۲ استفاده می‌کند.

### اولویت با اشتراک و fallback ارزان

```json
{
  "strategy": "priority",
  "targets": [
    { "provider": "claude-code-oauth", "weight": 1 },
    { "provider": "deepseek", "weight": 1 },
    { "provider": "kiro", "weight": 1 }
  ]
}
```

فهرست مرتب صریح مطابق سطح ۱ → سطح ۲ → سطح ۳.
