# راهنمای Progressive Web App (PWA)

OmniRoute به‌صورت یک Progressive Web App کاملاً قابل نصب عرضه می‌شود. هنگامی که از هر مرورگر موبایل — اندروید (Chrome) یا iOS (Safari) — به داشبورد دسترسی پیدا می‌کنید، می‌توانید «افزودن به صفحهٔ اصلی» را انتخاب کرده و یک تجربهٔ شبیه اپلیکیشن بومی بدون نیاز به فروشگاه اپلیکیشن دریافت کنید.

## PWA چیست؟

یک Progressive Web App داشبورد وب OmniRoute را به چیزی تبدیل می‌کند که ظاهر و احساسی شبیه اپلیکیشن موبایل بومی دارد. پس از نصب، این اپ:

- از صفحهٔ اصلی با آیکون اختصاصی خود اجرا می‌شود
- تمام‌صفحه باز می‌شود — بدون نوار آدرس مرورگر یا رابط کاربری زبانه
- با یک صفحهٔ اتصال اختصاصی به‌صورت آفلاین کار می‌کند
- دارایی‌های استاتیک را برای بارگذاری سریع‌تر کش می‌کند
- از هر دو حالت عمودی و افقی پشتیبانی می‌کند

## نصب

### اندروید (Chrome)

1. داشبورد OmniRoute را در Chrome باز کنید: `http://YOUR_IP:20128`
2. Chrome به‌صورت خودکار بنر **"Add OmniRoute to Home screen"** را نمایش می‌دهد، یا:
   - روی منوی **⋮** (سه نقطه) ضربه بزنید → **"Add to Home screen"** یا **"Install app"**
3. درخواست را تأیید کنید
4. OmniRoute به‌عنوان یک اپ مستقل در صفحهٔ اصلی شما ظاهر می‌شود

### iOS (Safari)

1. داشبورد OmniRoute را در Safari باز کنید: `http://YOUR_IP:20128`
2. روی دکمهٔ **Share** (جعبه با پیکان) ضربه بزنید
3. به پایین اسکرول کنید و روی **"Add to Home Screen"** ضربه بزنید
4. نام‌گذاری کنید (پیش‌فرض "OmniRoute") و روی **Add** ضربه بزنید
5. OmniRoute با آیکون اپ در صفحهٔ اصلی شما ظاهر می‌شود

### دسکتاپ (Chrome / Edge)

1. داشبورد OmniRoute را باز کنید
2. روی **آیکون نصب** در نوار آدرس کلیک کنید (یا ⋮ → "Install OmniRoute...")
3. درخواست را تأیید کنید
4. OmniRoute به‌صورت یک پنجرهٔ مستقل باز می‌شود — بدون زبانه، بدون نوار آدرس

## ویژگی‌ها

### تجربهٔ تمام‌صفحه

manifest با `display: "fullscreen"` پیکربندی شده است، یعنی اپ نصب‌شده از کل صفحه استفاده می‌کند — بدون رابط مرورگر، بدون همپوشانی نوار وضعیت. این امر باعث می‌شود داشبورد واقعاً بومی به‌نظر برسد.

### پشتیبانی آفلاین

OmniRoute شامل یک service worker (`sw.js`) است که کش هوشمند را ارائه می‌دهد:

| نوع دارایی                                               | استراتژی                        | رفتار                                                                                      |
| -------------------------------------------------------- | ------------------------------- | ------------------------------------------------------------------------------------------ |
| **App Shell**                                            | Cache-first                     | `/`، `/offline`، manifest و آیکون‌ها هنگام نصب از پیش کش می‌شوند                           |
| **دارایی‌های استاتیک** (CSS، JS، تصاویر، فونت‌ها)        | Network-first با fallback کش    | تازه را از شبکه واکشی می‌کند؛ در صورت آفلاین‌بودن به کش بازمی‌گردد                         |
| **باندل‌های Next.js** (`/_next/`)                        | Network-first با به‌روزرسانی کش | از شبکه واکشی و کش را به‌روزرسانی می‌کند؛ در صورت آفلاین‌بودن نسخهٔ کش‌شده را ارائه می‌دهد |
| **درخواست‌های ناوبری**                                   | Network-only با fallback آفلاین | همیشه از شبکه واکشی می‌کند؛ در صورت در دسترس نبودن شبکه صفحهٔ `/offline` را نمایش می‌دهد   |
| **مسیرهای API** (`/api/`، `/a2a`، `/dashboard/endpoint`) | دور زدن (هرگز کش نمی‌شود)       | همیشه مستقیماً به سرور می‌رود — هرگز توسط service worker رهگیری نمی‌شود                    |

### صفحهٔ آفلاین

هنگامی که شبکه در دسترس نیست و کاربر به یک صفحهٔ جدید می‌رود، service worker یک صفحهٔ اختصاصی `/offline` ارائه می‌دهد که:

- پیام روشن **«مشکل اتصال»** را نمایش می‌دهد
- یک **نشانگر وضعیت آنلاین/آفلاین زنده** نشان می‌دهد که در زمان واقعی به‌روزرسانی می‌شود
- یک دکمهٔ **«تلاش مجدد اتصال»** برای بارگذاری مجدد هنگام بازگشت اتصال ارائه می‌دهد
- به **صفحهٔ وضعیت** برای تشخیص پیوند می‌دهد

### آیکون‌های اپ

OmniRoute آیکون‌هایی بهینه‌شده برای هر پلتفرم ارائه می‌دهد:

| فایل                   | اندازه           | استفاده‌شده توسط               |
| ---------------------- | ---------------- | ------------------------------ |
| `icon-512.png`         | 512×512          | اعلان نصب اندروید، صفحهٔ اسپلش |
| `apple-touch-icon.png` | 180×180          | آیکون صفحهٔ اصلی iOS           |
| `icon-192.svg`         | 192×192 (برداری) | آیکون تطبیقی اندروید           |
| `apple-touch-icon.svg` | 180×180 (برداری) | fallback اپل                   |
| `favicon.svg`          | برداری           | زبانه‌های مرورگر               |
| `favicon.ico`          | چند‌اندازه       | مرورگرهای قدیمی                |

### ثبت خودکار

service worker به‌صورت خودکار از طریق کامپوننت `<PwaRegister />` در layout ریشه ثبت می‌شود. هیچ اقدامی از سوی کاربر لازم نیست — اپ به‌محض اینکه مرورگر manifest و service worker معتبر را تشخیص دهد، قابل نصب می‌شود.

## معماری فنی

### Web App Manifest (`manifest.webmanifest`)

توسط Next.js از طریق `src/app/manifest.ts` تولید می‌شود:

```json
{
  "name": "OmniRoute",
  "short_name": "OmniRoute",
  "description": "OmniRoute is an AI gateway for multi-provider LLMs. One endpoint for all your AI providers.",
  "start_url": "/",
  "scope": "/",
  "display": "fullscreen",
  "orientation": "any",
  "background_color": "#0b0f1a",
  "theme_color": "#0b0f1a",
  "icons": [
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png", "purpose": "any maskable" },
    { "src": "/apple-touch-icon.png", "sizes": "180x180", "type": "image/png" }
  ]
}
```

### Service Worker (`public/sw.js`)

یک service worker ساده (بدون وابستگی به فریم‌ورک) با:

- **مرحلهٔ نصب**: App shell را از پیش کش می‌کند (ریشه، صفحهٔ آفلاین، manifest، آیکون‌ها)
- **مرحلهٔ فعال‌سازی**: نسخه‌های قدیمی کش را پاک می‌کند و همهٔ کلاینت‌ها را claim می‌کند
- **مرحلهٔ fetch**: مسیریابی هوشمند بر اساس نوع درخواست (ناوبری، دارایی استاتیک، API)
- **نسخه‌گذاری کش**: `omniroute-pwa-v2` — برای اجبار کش تازه هنگام به‌روزرسانی این مقدار را افزایش دهید

### فرادادهٔ Layout (`src/app/layout.tsx`)

layout ریشه تمام متاتگ‌های لازم برای انطباق PWA را ارائه می‌دهد:

- پیوند `manifest` به `/manifest.webmanifest`
- `apple-web-app-capable: true` برای حالت مستقل iOS
- `apple-web-app-status-bar-style: black-translucent`
- `mobile-web-app-capable: yes` برای Android Chrome
- `theme-color: #0b0f1a`
- `viewport-fit: cover` برای رندر لبه‌به‌لبه

### کامپوننت: `PwaRegister`

این کامپوننت کلاینت در `src/shared/components/PwaRegister.tsx` قرار دارد:

1. هنگام mount اجرا می‌شود (فقط سمت کلاینت)
2. از پشتیبانی `serviceWorker` در مرورگر بررسی می‌کند
3. `/sw.js` را بی‌صدا ثبت می‌کند (خطاها برای جلوگیری از مسدودشدن اپ بلعیده می‌شوند)
4. چیزی رندر نمی‌کند (`return null`) — این یک کامپوننت فقط با عارضه جانبی است

## استفاده با Termux (اندروید)

هنگام اجرای OmniRoute روی اندروید از طریق Termux، PWA بی‌وقفه کار می‌کند:

1. OmniRoute را در Termux راه‌اندازی کنید: `npx omniroute`
2. Chrome را روی همان گوشی باز کنید: `http://localhost:20128`
3. PWA را از طریق «افزودن به صفحهٔ اصلی» نصب کنید
4. PWA به سرور محلی Termux متصل می‌شود — همه‌چیز روی دستگاه اجرا می‌شود

این ترکیب به این معناست که گوشی اندروید شما هم **سرور** (Termux) و هم **کلاینت** (PWA) است — یک دروازهٔ هوش مصنوعی کامل و خودکفا.

## استفاده از دستگاه‌های دیگر

PWA را روی هر دستگاهی که دسترسی مرورگر به سرور OmniRoute شما دارد نصب کنید:

- **گوشی/تبلت دیگر**: به `http://PHONE_IP:20128` بروید و PWA را نصب کنید
- **لپ‌تاپ**: Chrome/Edge را باز کنید و به‌عنوان PWA دسکتاپ نصبش کنید
- **تلویزیون هوشمند با مرورگر**: به داشبورد تمام‌صفحه دسترسی پیدا کنید

## سفارشی‌سازی

### نام نمونه

عنوان PWA از تنظیمات **نام نمونه** در `Dashboard → Settings` پیروی می‌کند. اگر نام نمونهٔ خود را به «My AI Gateway» تغییر دهید، PWA نصب‌شده آن نام را نمایش خواهد داد.

### favicon سفارشی

اگر یک favicon سفارشی از طریق `Dashboard → Settings` آپلود کنید، آیکون PWA روی دسکتاپ آیکون سفارشی را منعکس خواهد کرد. آیکون‌های صفحهٔ اصلی موبایل از فایل‌های از پیش ساخته‌شدهٔ `icon-512.png` و `apple-touch-icon.png` استفاده می‌کنند.

## محدودیت‌ها

- **بدون اعلان‌های push** — service worker از Push API پیاده‌سازی نمی‌کند. اعلان‌ها در عوض توسط اپ Electron مدیریت می‌شوند.
- **بدون همگام‌سازی پس‌زمینه** — اقدامات آفلاین برای پخش مجدد در صف قرار نمی‌گیرند. PWA عمدتاً یک نمایشگر داشبورد است.
- **محدودیت‌های iOS** — Safari روی iOS از همهٔ ویژگی‌های PWA پشتیبانی نمی‌کند (مثلاً اعلان‌های نصب دستی هستند و service worker‌های پس‌زمینه محدودند).
- **اندازهٔ کش** — service worker تنها دارایی‌های استاتیک را کش می‌کند. بارهای پاسخ بزرگ از مسیرهای `/api/` هرگز کش نمی‌شوند.
- **آیکون‌های سفارشی روی موبایل** — تغییر favicon در تنظیمات، آیکون صفحهٔ اصلی را روی موبایل به‌روزرسانی نمی‌کند (این کار بازتولید آیکون‌های PWA را می‌طلبد).
