# رتبه‌بندی ارائه‌دهنده‌های رایگان (Arena ELO)

> **خلاصه**: RouteChi ارائه‌دهنده‌های **رایگان** خود را بر اساس کیفیت مدل با استفاده از **امتیازهای ELO به سبک Arena AI
> (LMArena)** رتبه‌بندی می‌کند. صفحهٔ **رتبه‌بندی ارائه‌دهنده‌های رایگان** را در داشبورد باز کنید تا ببینید کدام ارائه‌دهنده‌های رایگان
> قوی‌ترین مدل‌ها را برای وظیفهٔ شما ارائه می‌دهند — به‌طور کلی، یا فیلترشده بر اساس دسته (کدنویسی، بازبینی، مستندسازی، رفع‌باگ).

---

## این صفحه چیست؟

RouteChi بیش از ۱۶۰ ارائه‌دهنده را تجمیع می‌کند که بسیاری از آن‌ها یک **سطح رایگان** دارند (بدون احراز هویت،
OAuth سطح رایگان، یا کلید API سطح رایگان — به
[راهنمای سطوح رایگان](../getting-started/FREE-TIERS-GUIDE.md) و فهرست کامل
[سطوح رایگان](../reference/FREE_TIERS.md) مراجعه کنید). نکتهٔ مهم: ارائه‌دهنده‌های رایگان از نظر کیفیت مدل
بسیار متنوع هستند. یک ارائه‌دهندهٔ بدون احراز هویت که یک مدل مرزی (frontier) ارائه می‌کند بسیار مفیدتر از
یک ارائه‌دهنده‌ای است که یک مدل قدیمی و کوچک سرو می‌دهد.

**رتبه‌بندی ارائه‌دهنده‌های رایگان** به این پرسش پاسخ می‌دهد که «**کدام ارائه‌دهندهٔ رایگان بهترین مدل را به من می‌دهد؟**»
با الحاق کاتالوگ هر ارائه‌دهندهٔ رایگان به **امتیازهای کیفیت مردمی** از
**تابلوی برترین‌های Arena AI** (ELO بر اساس ترجیح انسانی، همان ایدهٔ پشت
ساحت چت‌بات LMArena). سپس ارائه‌دهنده‌ها بر اساس قدرت **بهترین مدل رایگان** آن‌ها رتبه‌بندی می‌شوند.

رتبه‌بندی از سه منبع واقعی محاسبه می‌شود:

1. فهرست‌های ارائه‌دهندهٔ رایگان — `NOAUTH_PROVIDERS`، به‌علاوهٔ ورودی‌های `OAUTH_PROVIDERS` /
   `APIKEY_PROVIDERS` که با پرچم `hasFree` مشخص شده‌اند
   (`src/shared/constants/providers.ts`).
2. کاتالوگ مدل هر ارائه‌دهنده از رجیستری ارائه‌دهنده
   (`open-sse/config/providerRegistry.ts`).
3. امتیازهای تناسب وظیفهٔ مبتنی بر ELO که در جدول DB `model_intelligence` توسط
   موتور همگام‌سازی Arena ELO ذخیره می‌شوند (`src/lib/arenaEloSync.ts`).

منطق الحاق در `src/lib/freeProviderRankings.ts` قرار دارد.

## نحوهٔ دسترسی

### صفحهٔ داشبورد

داشبورد را باز کنید و به **Costs → Free Provider Rankings** بروید، یا مستقیماً به این نشانه بروید:

```
/dashboard/free-provider-rankings
```

این صفحه (`src/app/(dashboard)/dashboard/free-provider-rankings/page.tsx`) نمایش می‌دهد:

- **سکوی سه‌نفرهٔ برتر** (🥇 🥈 🥉) از بهترین ارائه‌دهنده‌های رایگان رتبه‌بندی‌شده.
- **جدول رتبه‌بندی کامل** با ستون‌های: **Rank**، **Provider**، **Top Model**،
  **Score**، **Avg Score**، **Models**، **Type**.
- **دکمه‌های فیلتر دسته**: _All Categories_، _Default_، _Coding_، _Review_،
  _Documentation_، _Debugging_.

نشان **Type** هر ارائه‌دهنده به شما می‌گوید که چگونه رایگان است:

| Badge    | Meaning                                       |
| -------- | --------------------------------------------- |
| `NOAUTH` | همیشه رایگان، بدون نیاز به اعتبارنامه         |
| `OAUTH`  | ارائه‌دهندهٔ OAuth با سطح رایگان (`hasFree`)    |
| `APIKEY` | ارائه‌دهندهٔ کلید API با سطح رایگان (`hasFree`) |

امتیازها به‌صورت برچسب‌های قابل‌خواندن توسط انسان (مانند _Elite_، _Excellent_، _Very Good_،
_Good_، _Average_) نمایش داده می‌شوند نه اعداد خام، زیرا مقدار زیرین یک کیفیت
رتبه‌بندی نسبی است، نه درصد.

### نقطهٔ پایانی API

این صفحه توسط یک نقطهٔ پایانی خواندنی عمومی پشتیبانی می‌شود
(`src/app/api/free-provider-rankings/route.ts`):

```
GET /api/free-provider-rankings
GET /api/free-provider-rankings?category=coding
GET /api/free-provider-rankings?category=coding&limit=20
```

پارامترهای کوئری (با Zod اعتبارسنجی می‌شوند):

| Param      | Type   | Default | Notes                                                                                              |
| ---------- | ------ | ------- | -------------------------------------------------------------------------------------------------- |
| `category` | string | (none)  | یکی از `default`، `coding`، `review`، `documentation`، `debugging`. برای رتبه‌بندی ترکیبی حذف کنید. |
| `limit`    | number | `50`    | در محدودهٔ `1–100` محدود می‌شود.                                                                      |

شکل پاسخ:

```json
{
  "rankings": [
    {
      "id": "<provider-id>",
      "name": "<provider name>",
      "icon": "<icon>",
      "color": "<hex color>",
      "textIcon": "<short label>",
      "category": "noauth | oauth | apikey",
      "topModel": {
        "modelId": "<registry model id>",
        "modelName": "<model display name>",
        "score": 0.0,
        "eloRaw": 0,
        "confidence": "high | medium | low",
        "category": "<task category>"
      },
      "averageScore": 0.0,
      "modelCount": 0
    }
  ]
}
```

`eloRaw` مقدار اصلی Arena ELO است؛ `score` مقدار نرمال‌شدهٔ تناسب وظیفه است
(در ادامه ببینید). ارائه‌دهنده‌هایی که مدل امتیازداری ندارند از پاسخ حذف می‌شوند.

## نحوهٔ کار امتیازها

### منبع: تابلوی برترین‌های Arena AI

موتور همگام‌سازی Arena ELO (`src/lib/arenaEloSync.ts`) دو تابلو را — `text`
و `code` — از API تابلوی برترین‌های Arena AI
(`https://api.wulong.dev/arena-ai-leaderboards/v1/leaderboard`) دریافت می‌کند. هر ورودی تابلو
شامل نام مدل، سازنده، `score` مربوط به ELO، بازهٔ اطمینان و تعداد رأی است.

دسته‌های تابلو به دسته‌های وظیفهٔ RouteChi نگاشت می‌شوند:

| Arena leaderboard | RouteChi task categories                          |
| ----------------- | ------------------------------------------------- |
| `text`            | `default`، `review`، `documentation`، `debugging` |
| `code`            | `coding`                                          |

### نرمال‌سازی (امتیاز تناسب وظیفه)

امتیازهای خام ELO به‌ازای هر تابلو به یک **مقدار تناسب وظیفه در `[0.4, 0.98]`** نرمال می‌شوند:

```
taskFit = 0.4 + 0.58 * ((elo - minElo) / (maxElo - minElo))
```

این امتیاز هرگز به `0` یا `1` نمی‌رسد و فضایی برای نادیده‌گیری‌های کاربر باقی می‌گذارد. این همان
فیلد `score` است که در پاسخ API می‌بینید و برچسبی است که در داشبورد نمایش داده می‌شود.

### اطمینان

هر ورودی بر اساس تعداد رأی Arena یک سطح اطمینان دریافت می‌کند:

| Confidence | Votes   |
| ---------- | ------- |
| `high`     | ≥ ۵٬۰۰۰ |
| `medium`   | ≥ ۱٬۰۰۰ |
| `low`      | < ۱٬۰۰۰ |

### ذخیره‌سازی و تازگی

ورودی‌های نرمال‌شده به جدول DB `model_intelligence` با
`source = "arena_elo"` نوشته می‌شوند (`src/lib/db/modelIntelligence.ts`). ورودی‌ها **پس از
۷ روز منقضی می‌شوند**، بنابراین ارائه‌دهنده‌ای که همگام‌سازی را متوقف می‌کند در نهایت
حذف می‌شود تا داده‌های کهنه سرو نکند.

همگام‌سازی **به‌طور پیش‌فرض فعال** است:

- یک‌بار در زمان راه‌اندازی سرور اجرا می‌شود و سپس روی تایمر دوره‌ای
  (`src/lib/arenaEloSync.ts`، متصل از `src/server-init.ts`).
- **غیرمسدودکننده و هرگز کشنده** است — اگر دریافت از بالادست ناموفق شود، RouteChi به
  کار ادامه می‌دهد و رتبه‌بندی‌ها صرفاً آخرین دادهٔ خوب را (یا یک حالت خالی) نمایش می‌دهند.

دو متغیر محیطی آن را کنترل می‌کنند (در
[`docs/reference/ENVIRONMENT.md`](../reference/ENVIRONMENT.md) مستند شده‌اند):

| Variable                  | Default       | Purpose                                         |
| ------------------------- | ------------- | ----------------------------------------------- |
| `ARENA_ELO_SYNC_ENABLED`  | `true`        | روی `false` تنظیم کنید تا از همگام‌سازی خروجی انصراف دهید. |
| `ARENA_ELO_SYNC_INTERVAL` | `86400` (24h) | بازهٔ همگام‌سازی، به ثانیه.                      |

### همگام‌سازی دستی / وضعیت / پاک‌سازی

برای اپراتورها، یک نقطهٔ پایانی مدیریتِ احراز هویت‌شده کنترل دستی را فراهم می‌کند
(`src/app/api/intelligence/sync/route.ts` — نیازمند احراز هویت مدیریت):

```
GET    /api/intelligence/sync          # current sync status (enabled, lastSync, nextSync, intervalMs)
POST   /api/intelligence/sync          # trigger a manual sync; body: { "dryRun": true } to preview without writing
DELETE /api/intelligence/sync          # clear all synced arena_elo intelligence entries
```

اگر صفحهٔ رتبه‌بندی خالی است، یک `POST /api/intelligence/sync` دستی (یا صرفاً
راه‌اندازی مجدد سرور) آن را مجدداً پر می‌کند.

### تطبیق مدل‌ها با تابلو

شناسه‌های مدل رجیستری و نام‌های مدل Arena همیشه دقیقاً مطابقت ندارند. رتبه‌بندی از
تطبیق انعطاف‌پذیر استفاده می‌کند (`findMatchingIntelligence` در `src/lib/freeProviderRankings.ts`):

1. تطبیق دقیق روی شناسهٔ مدل نرمال‌شده.
2. تطبیق پس از حذف پسوند نسخهٔ انتهایی (مثلاً `kimi-k2.6` ← `kimi-k2`).
3. تطبیق پیشوند (نام مدل تابلو پیشوند شناسهٔ رجیستری باشد).

در سمت همگام‌سازی، پیشوندهای سازندهٔ شناخته‌شده (`anthropic/`، `openai/`، `google/`، …)
حذف می‌شوند و یک نقشهٔ نام‌های مستعار کوچک، نام‌های رسمی را به انواع متغیرهایی که RouteChi به‌طور
داخلی استفاده می‌کند بسط می‌دهد، تا مدل‌ها تحت هر نامی قابل‌یافت بمانند.

### نحوهٔ رتبه‌بندی یک ارائه‌دهنده

برای هر ارائه‌دهندهٔ رایگان، موتور هر مدل در کاتالوگش را امتیازدهی می‌کند، سپس:

- **Top Model** = بالاترین امتیاز مدلِ ارائه‌دهنده.
- **Avg Score** = میانگین امتیاز در سراسر تمام مدل‌های امتیازداری‌شدهٔ آن ارائه‌دهنده.
- **Models** = تعداد مدل‌های ارائه‌دهنده که امتیاز Arena داشته‌اند.

ارائه‌دهنده‌ها ابتدا بر اساس **امتیاز مدل برتر**، سپس بر اساس میانگین امتیاز مرتب می‌شوند. این کار
ارائه‌دهنده‌ای را پاداش می‌دهد که حداقل یک مدل رایگان قوی سرو می‌کند.

## استفاده برای انتخاب ارائه‌دهنده‌های رایگان

1. **دستهٔ درست را انتخاب کنید.** برای بارهای کاری عاملیانه/کد از فیلتر **Coding** استفاده کنید، یا
   برای گفت‌وگوی عمومی روی **All Categories** / **Default** بگذارید. ممکن است یک ارائه‌دهندهٔ واحد
   در دسته‌های مختلف رتبه‌های متفاوتی داشته باشد زیرا مدل برتر آن به‌ازای هر تابلو متفاوت است.
2. **برای راه‌اندازی‌های یک‌باره، سکوی برتر را ترجیح دهید.** اگر می‌خواهید تنها یک یا دو ارائه‌دهندهٔ رایگان
   متصل کنید، با رتبه‌های برتر برای دستهٔ خود شروع کنید.
3. **نشان Type را بررسی کنید.** ارائه‌دهنده‌های `NOAUTH` سریع‌ترین برای اتصال هستند (بدون
   اعتبارنامه). سطوح رایگان `OAUTH` / `APIKEY` نیاز به یک ثبت‌نام سریع دارند اما اغلب
   مدل‌های قوی‌تری را عرضه می‌کنند. برای
   مراحل اتصال به [راهنمای سطوح رایگان](../getting-started/FREE-TIERS-GUIDE.md) مراجعه کنید.
4. **چند مورد را متصل کنید و بگذارید Auto-Combo تصمیم بگیرد.** همان دادهٔ Arena ELO که
   این صفحه را نیرو می‌بخشد، **عامل تناسب وظیفه** را نیز در موتور امتیازدهی Auto-Combo تغذیه می‌کند
   (`open-sse/services/autoCombo/taskFitness.ts`، ترتیب تفکیک
   `user_override → arena_elo → models_dev_tier → static table`). بنابراین پس از اتصال
   ارائه‌دهنده‌های رایگان برتر، مسیریابی با `model: "auto"` (مثلاً `auto/coding`) به‌طور
   خودکار مدل‌های رایگان با کیفیت بالاتر را در هر درخواست ترجیح خواهد داد. برای
   امتیازدهی کامل ۹عاملی به [Auto-Combo](../routing/AUTO-COMBO.md) مراجعه کنید.

## مستندات مرتبط

- [راهنمای سطوح رایگان](../getting-started/FREE-TIERS-GUIDE.md) — نحوهٔ اتصال ارائه‌دهنده‌های
  رایگان، بدون نیاز به کارت اعتباری.
- [فهرست سطوح رایگان](../reference/FREE_TIERS.md) — کاتالوگ کامل ارائه‌دهنده‌های رایگان
  و محدودیت‌های آن‌ها.
- [Auto-Combo](../routing/AUTO-COMBO.md) — موتور مسیریابی ۹عاملی که همان
  دادهٔ تناسب وظیفهٔ Arena ELO را مصرف می‌کند.
- [متغیرهای محیطی](../reference/ENVIRONMENT.md) — مرجع `ARENA_ELO_SYNC_ENABLED` /
  `ARENA_ELO_SYNC_INTERVAL`.
