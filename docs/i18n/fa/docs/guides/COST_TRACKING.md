# پیگیری هزینه و مصرف

نحوهٔ تخمین، ثبت و گزارش هزینهٔ هر درخواست توسط OmniRoute — و چرا عدد داشبورد یک
**ردیاب صرفه‌جویی** است، نه یک صورت‌حساب.

همچنین ببینید: [راهنمای کاربر](./USER_GUIDE.md) · [گالری ویژگی‌ها](./FEATURES.md)

---

## این چیست (و چه چیزی نیست)

OmniRoute با ضرب تعداد توکن در نرخ‌های قیمت‌گذاری یک مدل، هزینهٔ دلاری به ازای هر درخواست
را به هر completion نسبت می‌دهد. این اعداد داشبورد **Costs**، CLI مربوط به
`omniroute cost` / `omniroute usage`، صادرات CSV/JSON و بودجه‌های به‌ازای کلید API را
تغذیه می‌کنند.

> **"هزینه" داشبورد یک ردیاب صرفه‌جویی است، نه یک صورت‌حساب.** OmniRoute هرگز از شما charge
> نمی‌کند — درخواست‌های شما را به ارائه‌دهنده‌هایی که از قبل متصل کرده‌اید (اشتراک‌های خود،
> سطوح رایگان و کلیدهای API خود) مسیریابی می‌کند. یک "هزینهٔ کل ۲۹۰ دلاری" که کاملاً روی
> مدل‌های رایگان انباشته شده به این معنی است که تقریباً **۲۹۰ دلاری که به یک API پرداختی
> _نکردید_**. این عدد یک _تخمین_ از هزینهٔ ترافیک مشابه در قیمت‌های فهرستی استاندارد است،
> تا ببینید استفادهٔ شما کجا متمرکز است و مسیریابی به ارائه‌دهنده‌های ارزان‌تر/رایگان چقدر
> برایتان صرفه‌جویی کرده است.

این چارچوب مستقیماً در [README](../../README.md) پروژه بیان شده ("هزینهٔ داشبورد یک ردیاب
صرفه‌جویی است، نه یک صورت‌حساب").

چون این عدد یک تخمین است:

- به جدول قیمت‌گذاری که OmniRoute برای هر مدل دارد بستگی دارد. یک مدل بدون ورودی قیمت‌گذاری
  هزینهٔ `0` تولید می‌کند (به‌عنوان یک ردیف "Legacy / Free" در explorer نمایش داده می‌شود).
- ترافیک سطح رایگان و اشتراک همچنان یک هزینهٔ _تخمینی_ انباشته می‌کند — این مبلغی است که
  صرفه‌جویی می‌کنید، نه مبلغی که بدهکار هستید.

## نحوهٔ تخمین هزینه‌ها

### منبع قیمت‌گذاری

هزینه‌ها از یک جدول قیمت‌گذاری با این ترتیب تقدم تحلیل می‌شوند
([`src/lib/pricingSync.ts`](../../src/lib/pricingSync.ts)):

1. **بازنویسی‌های کاربر** — قیمتی که در داشبورد / از طریق `PATCH /api/pricing` تنظیم کرده‌اید.
2. **قیمت‌گذاری خارجی همگام‌شده** — واکشی از `model_prices_and_context_window.json` عمومی
   LiteLLM هنگامی که همگام‌سازی فعال است (در یک namespace جداگانهٔ `pricing_synced` ذخیره
   می‌شود تا هرگز بازنویسی‌های شما را بازنویسی نکند).
3. **پیش‌فرض‌های hardcoded** — همراه OmniRoute ارائه می‌شوند.

همگام‌سازی قیمت‌گذاری خارجی **opt-in** است و به‌طور پیش‌فرض غیرفعال است. متغیرهای env مرتبط
(به [`.env.example`](../../.env.example) مراجعه کنید):

| متغیر env               | پیش‌فرض   | هدف                                                                 |
| ----------------------- | --------- | ------------------------------------------------------------------- |
| `PRICING_SYNC_ENABLED`  | `false`   | فعال‌سازی همگام‌سازی پس‌زمینهٔ قیمت‌گذاری LiteLLM هنگام راه‌اندازی. |
| `PRICING_SYNC_INTERVAL` | `86400`   | فاصلهٔ همگام‌سازی به **ثانیه** (پیش‌فرض روزانه).                    |
| `PRICING_SYNC_SOURCES`  | `litellm` | فهرست منبع جدا‌شده با کاما (امروزه تنها `litellm` پشتیبانی می‌شود). |

### فرمول هزینه

هزینه به ازای هر درخواست از تعداد توکن و نرخ‌های به‌ازای-میلیون-توکن در
[`src/lib/usage/costCalculator.ts`](../../src/lib/usage/costCalculator.ts)
محاسبه می‌شود (`computeCostFromPricing` / `calculateCost`):

- **توکن‌های ورودی** (کسر cache readها و توکن‌های cache-creation) × نرخ `input`.
- **توکن‌های cache-read** × نرخ `cached` (به نرخ ورودی fallback می‌شود).
- **توکن‌های cache-creation** × نرخ `cache_creation` (به نرخ ورودی fallback می‌شود).
- **توکن‌های خروجی** × نرخ `output`.
- **توکن‌های reasoning** × نرخ `reasoning` (به نرخ خروجی fallback می‌شود).

تمام نرخ‌ها به‌عنوان دلار به ازای ۱٬۰۰۰٬۰۰۰ توکن تفسیر می‌شوند. یک service tier مربوط به
Codex با "fast"/"priority" یا "flex" یک ضریب هزینه اعمال می‌کند (`getCodexFastCostMultiplier`)
— مثلاً flex با تخفیف ۵۰٪ توکن صورت‌حساب می‌شود و به‌عنوان **flex savings** در داشبورد نمایش
داده می‌شود.

نام‌های مدل ابتدا نرمالایز می‌شوند (پیشوندهای مسیر ارائه‌دهنده مانند `openai/` یا
`accounts/fireworks/models/` حذف می‌شوند) تا ردیف‌های تاریخی همچنان با یک قیمت تطابق داشته باشند.

### نحوهٔ ثبت مصرف

- هزینهٔ به‌ازای هر درخواست پس از پاسخ محاسبه و fire-and-forget ثبت می‌شود تا هرگز latency
  به کلاینت اضافه نکند. مصرف سهمیهٔ مشترک در tick بعدی event-loop از طریق
  [`src/lib/quota/spendRecorder.ts`](../../src/lib/quota/spendRecorder.ts) زمان‌بندی می‌شود.
- مصرف کلید API توسط
  [`SpendBatchWriter`](../../src/lib/spend/batchWriter.ts) بافر و به دسته‌ها flush می‌شود
  (فاصلهٔ flush پیش‌فرض 60s، بافر ۱٬۰۰۰ ورودی). قابل تنظیم از طریق:

  | متغیر env                           | پیش‌فرض | هدف                                    |
  | ----------------------------------- | ------- | -------------------------------------- |
  | `OMNIROUTE_SPEND_FLUSH_INTERVAL_MS` | `60000` | فاصلهٔ flush به میلی‌ثانیه.            |
  | `OMNIROUTE_SPEND_MAX_BUFFER_SIZE`   | `1000`  | حداکثر ورودی‌های بافرشده پیش از flush. |

اعداد هزینهٔ داشبورد از یک مبلغ دلاری ذخیره‌شده به‌ازای ردیف **خوانده نمی‌شوند** — آنها هر
بار که نقطهٔ پایانی تحلیل اجرا می‌شود، در پرواز از تعداد توکن و جدول قیمت‌گذاری فعلی
دوباره محاسبه می‌شوند. این بدان معناست که اصلاح یک قیمت اشتباه (و همگام‌سازی مجدد) تخمین‌های
هزینهٔ تاریخی را به‌طور retroactive به‌روزرسانی می‌کند.

## داشبورد: صفحهٔ Costs

صفحهٔ **Costs** در `/dashboard/costs`
(`src/app/(dashboard)/dashboard/costs/`) قرار دارد.
نمای اصلی آن تب **Cost Overview** است
(`src/app/(dashboard)/dashboard/costs/CostOverviewTab.tsx`)،
که همه‌چیز را از `GET /api/usage/analytics` بارگذاری می‌کند.

آنچه نشان می‌دهد:

- **کاشی‌های مصرف** — مصرف تخمینی برای _Today (1d)_، _7d_، _30d_ و پنجرهٔ انتخاب‌شده.
  انتخابگر بازه: `7d`، `30d`، `90d`، `all`.
- **معیارهای تیتر** — درخواست‌ها در پنجره، ارائه‌دهنده‌های فعال، مدل‌های فعال، میانگین
  هزینه به ازای درخواست.
- **Cost Explorer** — یک جدول قابل مرتب‌سازی/فیلتر گروه‌بندی‌شده بر اساس **ارائه‌دهنده**،
  **مدل**، **کلید API**، **حساب** یا **service tier**، با هزینه، درخواست‌ها، توکن، میانگین
  هزینه/درخواست و درصد سهم-از-کل.
- **استفادهٔ توکن** — مجموع توکن‌های ورودی / خروجی / کل و نسبت ورودی:خروجی.
- **کارایی مسیریابی** — تعداد fallback، نرخ fallback و پوشش مدل-درخواست‌شده.
- **پیش‌بینی ماهانه** — هزینهٔ پایان ماه را از میانگین روزانه اخیر projection می‌کند.
- **مقایسهٔ دوره** — تغییر درصدی بین نیمهٔ اول و دوم پنجره.
- **نمودارها** — روند هزینهٔ روزانه، سهم ارائه‌دهنده (دایره‌ای)، برترین ارائه‌دهنده‌ها،
  برترین مدل‌ها، هزینه بر اساس کلید API، هزینه بر اساس حساب، الگوی استفادهٔ هفتگی و یک
  نقشهٔ حرارتی فعالیت.
- **صادرات** — دانلود پنجرهٔ فعلی به‌صورت **CSV** یا **JSON** (دکمه‌ها پس از وجود دادهٔ
  هزینهٔ غیرصفر ظاهر می‌شوند).

هنگامی که ترافیک قیمت‌گذاری‌شده‌ای وجود ندارد، ردیف‌ها به‌جای `$0` با یک برچسب "Legacy / Free"
رندر می‌شوند که مدل ردیاب صرفه‌جویی را منعکس می‌کند.

### زیرصفحه‌های مرتبط Costs

ناحیهٔ Costs همچنین میزبان (همه تحت `/dashboard/costs/`):

- **Pricing** (`/dashboard/costs/pricing`) — مشاهده و بازنویسی قیمت‌های به‌ازای مدل (تب
  مشترک Pricing را رندر می‌کند).
- **Budget** (`/dashboard/costs/budget`) — تنظیم محدودیت‌های مصرف به‌ازای-scope (تب مشترک
  Budget را رندر می‌کند).
- **Quota Share** (`/dashboard/costs/quota-share`) — poolهای سهمیهٔ مشترک و نماهای burn-rate.

## نقاط پایانی API

تمام این موارد نیازمند auth مدیریتی هستند (loopback/JWT، از طریق `requireManagementAuth`) مگر
خلاف آن ذکر شده باشد.

### تحلیل استفاده و هزینه

| متد   | نقطهٔ پایانی             | هدف                                                                                                                                                       |
| ----- | ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `GET` | `/api/usage/analytics`   | تحلیل کامل هزینه/استفاده: خلاصه، روند روزانه، به ازای ارائه‌دهنده/مدل/کلید API/حساب/tier. Query: `range`، `startDate`، `endDate`، `apiKeyIds`، `presets`. |
| `GET` | `/api/usage/utilization` | استفادهٔ سهمیهٔ به‌ازای ارائه‌دهنده در طول زمان. Query: `range` (`1h`/`24h`/`7d`/`30d`)، `provider`.                                                      |
| `GET` | `/api/usage/history`     | ردیف‌های خام تاریخچهٔ استفاده.                                                                                                                            |
| `GET` | `/api/usage/call-logs`   | لاگ‌های call به‌ازای درخواست (مدل، توکن، هزینه، latency، وضعیت).                                                                                          |
| `GET` | `/api/usage/quota`       | وضعیت سهمیهٔ ارائه‌دهنده.                                                                                                                                 |
| `GET` | `/api/usage/proxy-logs`  | لاگ‌های درخواست پروکسی.                                                                                                                                   |

### بودجه‌ها

| متد    | نقطهٔ پایانی             | هدف                                                                                |
| ------ | ------------------------ | ---------------------------------------------------------------------------------- |
| `GET`  | `/api/usage/budget`      | خلاصهٔ هزینه + بررسی بودجه برای یک کلید API (پارامتر query `apiKeyId` الزامی است). |
| `POST` | `/api/usage/budget`      | تنظیم محدودیت‌های USD روزانه/هفتگی/ماهانه + آستانهٔ هشدار برای یک کلید API.        |
| `GET`  | `/api/usage/budget/bulk` | خلاصه‌های bulk بودجه در میان کلیدهای API.                                          |

> API بودجه به ازای **کلید API** (`apiKeyId`) scoped می‌شود. محدودیت‌های بازگردانده‌شده توسط
> `GET /api/usage/budget` شامل `dailyLimitUsd`، `weeklyLimitUsd`، `monthlyLimitUsd`،
> یک `warningThreshold` و کلهای در حال اجرا (`totalCostToday`، `totalCostMonth`، …) است.

### قیمت‌گذاری

| متد      | نقطهٔ پایانی            | هدف                                                                                                           |
| -------- | ----------------------- | ------------------------------------------------------------------------------------------------------------- |
| `GET`    | `/api/pricing`          | قیمت‌گذاری mergeشدهٔ فعلی (کاربر + همگام‌شده + پیش‌فرض‌ها). `?includeSources=1` برای دیدن منبع به ازای ورودی. |
| `PATCH`  | `/api/pricing`          | بازنویسی قیمت‌گذاری برای `{ provider: { model: { input, output, cached, … } } }`.                             |
| `DELETE` | `/api/pricing`          | بازنشانی قیمت‌گذاری به پیش‌فرض‌ها (اختیاری scoped توسط `?provider=&model=`).                                  |
| `GET`    | `/api/pricing/defaults` | نمایش نرخ‌های fallback پیش‌فرض به‌ازای-1M.                                                                    |
| `GET`    | `/api/pricing/models`   | قیمت‌گذاری keyed شده بر اساس مدل.                                                                             |
| `POST`   | `/api/pricing/sync`     | راه‌اندازی یک همگام‌سازی دستی از منابع خارجی (LiteLLM).                                                       |
| `GET`    | `/api/pricing/sync`     | وضعیت همگام‌سازی فعلی.                                                                                        |
| `DELETE` | `/api/pricing/sync`     | پاک‌سازی تمام دادهٔ قیمت‌گذاری همگام‌شده.                                                                     |

### سایر نقاط پایانی مرتبط با هزینه

| متد   | نقطهٔ پایانی                  | هدف                                                                       |
| ----- | ----------------------------- | ------------------------------------------------------------------------- |
| `GET` | `/api/free-tier/summary`      | مجموع توکن مدل رایگان، استفاده‌شده-این-ماه و allowance رایگان باقی‌مانده. |
| `GET` | `/api/quota/pools/[id]/usage` | استفاده برای یک pool سهمیهٔ مشترک.                                        |

## CLI

CLI مربوط به OmniRoute دستورات هزینه، استفاده و قیمت‌گذاری را عرضه می‌کند (ثبت‌شده در
[`bin/cli/commands/registry.mjs`](../../bin/cli/commands/registry.mjs)).

### `omniroute cost`

یک گزارش هزینه تجمیع‌شده از `/api/usage/analytics`.

```bash
omniroute cost                          # last 30d, grouped by provider
omniroute cost --period 7d              # last 7 days
omniroute cost --group-by model         # group by provider | model | combo | api-key | day
omniroute cost --since 2026-06-01 --until 2026-06-13
omniroute cost --api-key <key> --limit 50
```

ستون‌ها: group، درخواست‌ها، توکن ورودی/خروجی، هزینه (USD) و ٪ از کل. یک خط grand total
در انتها چاپ می‌شود (با `--quiet` یا `--output json` suppressed می‌شود).

### `omniroute usage`

```bash
omniroute usage analytics --period 30d [--provider <id>]   # per-provider cost summary
omniroute usage logs [--limit 100] [--follow] [--api-key <k>] [--search <q>]
omniroute usage quota [--provider <id>] [--check]
omniroute usage utilization [--api-key <k>]
omniroute usage history [--limit 100]
omniroute usage proxy-logs [--limit 100]

# Budgets
omniroute usage budget list
omniroute usage budget get [scope]
omniroute usage budget set <amount> [--scope global] [--period monthly]
omniroute usage budget reset [scope]
```

### `omniroute pricing`

```bash
omniroute pricing list [--provider <p>] [--model <m>] [--limit 200]
omniroute pricing get <model>
omniroute pricing sync [--provider <p>] [--force]   # POST /api/pricing/sync
omniroute pricing diff [--model <m>]
omniroute pricing defaults show
omniroute pricing defaults set [--input <p>] [--output <p>] [--cache-read <p>] [--cache-write <p>]
```

> `pricing defaults show` از `GET /api/pricing/defaults` می‌خواند. برای ویرایش قیمت‌های
> مدل‌های منفرد به‌جای آن از صفحهٔ داشبورد **Pricing** یا `PATCH /api/pricing` استفاده کنید.

## رفع اشکال

- **تمام هزینه‌ها $0 / "Legacy / Free" را نشان می‌دهند.** مدل‌های در حال استفاده ورودی
  قیمت‌گذاری ندارند. همگام‌سازی خارجی را فعال کنید (`PRICING_SYNC_ENABLED=true`) و
  `omniroute pricing sync` را اجرا کنید، یا قیمت‌ها را به‌صورت دستی از طریق صفحهٔ Pricing /
  `PATCH /api/pricing` تنظیم کنید.
- **یک مدل تاریخی اشتباه قیمت‌گذاری شده.** قیمت را اصلاح کنید (بازنویسی یا همگام‌سازی مجدد)
  — هزینه در هر خواندن تحلیل از تعداد توکن دوباره محاسبه می‌شود، بنابراین تخمین‌ها retroactive
  به‌روزرسانی می‌شوند.
- **مصرفی از زمان واقعی عقب می‌افتد.** مصرف به‌ازای کلید دسته‌ای است؛ اگر به اعداد تازه‌تری
  نیاز دارید `OMNIROUTE_SPEND_FLUSH_INTERVAL_MS` را کاهش دهید.
