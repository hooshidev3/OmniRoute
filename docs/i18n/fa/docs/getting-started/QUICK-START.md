# شروع سریع: RouteChi را در ۳ دقیقه راه‌اندازی کنید

> **خلاصه**: نصب → اتصال به یک ارائه‌دهنده رایگان → IDE خود را به RouteChi متصل کنید. تمام.

---

## گام ۱: نصب RouteChi

روش دلخواه خود را انتخاب کنید:

### گزینه A: npm (پیشنهادی)

```bash
npm install -g routechi
```

### گزینه B: Docker

```bash
docker run -d --name omniroute -p 20128:20128 borhandarabi/routechi:latest
```

### گزینه C: از روی سورس

```bash
git clone https://github.com/borhandarabi/routechi.git
cd RouteChi
npm install
npm run dev
```

---

## گام ۲: راه‌اندازی RouteChi

```bash
omniroute
```

RouteChi در نشانی `http://localhost:20128` شروع به کار می‌کند. داشبورد به‌صورت خودکار باز می‌شود.

---

## گام ۳: اتصال به یک ارائه‌دهنده رایگان

می‌توانید با اتصال به یک ارائه‌دهنده رایگان، **بدون پرداخت هیچ هزینه‌ای** از RouteChi استفاده کنید.

### گزینه A: Kiro (کلود رایگان — بدون کارت اعتباری)

1. داشبورد را در `http://localhost:20128` باز کنید
2. به **Providers** → **Add Provider** بروید
3. **Kiro AI** را انتخاب کنید
4. روی **Connect** کلیک کنید (بدون نیاز به کلید API!)
5. تمام! حالا به مدل‌های کلود دسترسی رایگان دارید.

### گزینه B: OpenCode Free (بدون احراز هویت)

1. داشبورد را در `http://localhost:20128` باز کنید
2. به **Providers** → **Add Provider** بروید
3. **OpenCode Free** را انتخاب کنید
4. روی **Connect** کلیک کنید (بدون نیاز به کلید API!)
5. تمام! حالا به چندین مدل دسترسی رایگان دارید.

### گزینه C: Pollinations (بدون نیاز به کلید)

1. داشبورد را در `http://localhost:20128` باز کنید
2. به **Providers** → **Add Provider** بروید
3. **Pollinations** را انتخاب کنید
4. روی **Connect** کلیک کنید (بدون نیاز به کلید API!)
5. تمام! حالا به GPT-5، Claude، Gemini و موارد دیگر دسترسی رایگان دارید.

---

## گام ۴: IDE خود را به RouteChi متصل کنید

در IDE یا ابزار CLI خود، این مقادیر را تنظیم کنید:

```
Base URL: http://localhost:20128/v1
API Key:  [از Dashboard → Endpoints کپی کنید]
Model:    auto
```

این تمام کاری بود که باید بکنید! حالا IDE شما با انتخاب خودکار ارائه‌دهنده از RouteChi استفاده می‌کند.

---

## گام ۵: بررسی کارکرد

```bash
curl http://localhost:20128/v1/models -H "Authorization: Bearer YOUR_KEY"
```

باید لیست مدل‌های متصل خود را ببینید.

## سؤالات متداول

### «آیا به کلید API نیاز دارم؟»

**خیر!** می‌توانید از ارائه‌دهندگان رایگان (Kiro، OpenCode Free، Pollinations) بدون هیچ کلید API استفاده کنید. کافیست آن‌ها را در داشبورد متصل کنید.

### «`auto` چیست؟»

`auto` به RouteChi می‌گوید که برای هر درخواست بهترین ارائه‌دهنده را به‌صورت خودکار انتخاب کند. این سیستم سرعت، هزینه، کیفیت و در دسترس بودن را در نظر می‌گیرد. برای جزئیات به [راهنمای کامبو خودکار](./AUTO-COMBO-GUIDE.md) مراجعه کنید.

### «چقدر هزینه دارد؟»

خود RouteChi **رایگان و متن‌باز** است. شما فقط برای ارائه‌دهندگانی که استفاده می‌کنید هزینه می‌پردازید. بسیاری از ارائه‌دهندگان سطوح رایگان دارند — به [راهنمای سطوح رایگان](./FREE-TIERS-GUIDE.md) مراجعه کنید.

### «آیا می‌توانم با Claude Code / Cursor / Copilot استفاده کنم؟»

**بله!** RouteChi با هر ابزاری که از فرمت OpenAI پشتیبانی کند کار می‌کند. کافیست base URL را روی `http://localhost:20128/v1` تنظیم کنید. برای دستورالعمل‌های راه‌اندازی خاص به [راهنمای ابزارهای CLI](../reference/CLI-TOOLS.md) مراجعه کنید.

### «اگر یک ارائه‌دهنده از کار بیفتد چه؟**

RouteChi به‌صورت خودکار ارائه‌دهندگان ناموفق را رد می‌کند و گزینه بعدی را امتحان می‌کند. نیازی به کاری ندارید. برای جزئیات به [راهنمای کامبو خودکار](./AUTO-COMBO-GUIDE.md) مراجعه کنید.
