# کامبو خودکار: بگذارید RouteChi بهترین هوش مصنوعی را برای شما انتخاب کند

> **خلاصه**: کافیست مدل خود را روی `auto` تنظیم کنید تا RouteChi برای هر درخواست بهترین ارائه‌دهنده هوش مصنوعی را به‌صورت خودکار انتخاب کند. نیازی به پیکربندی نیست.

---

## این ویژگی چه می‌کند؟

به‌جای انتخاب یک مدل خاص هوش مصنوعی (مثل GPT-4o یا Claude)، می‌توانید بگذارید RouteChi **به‌صورت خودکار بهترین گزینه** را برای هر درخواست انتخاب کند. این سیستم به این موارد توجه می‌کند:

- **سلامت** — آیا ارائه‌دهنده در حال حاضر کار می‌کند؟
- **سرعت** — چقدر سریع است؟
- **هزینه** — چقدر هزینه دارد؟
- **کیفیت** — آیا در این نوع وظیفه خوب عمل می‌کند؟
- **ظرفیت** — آیا سهمیه باقی‌مانده دارد؟

RouteChi تمام ارائه‌دهندگان متصل شما را امتیازدهی می‌کند و بهترین را انتخاب می‌کند. اگر شکست بخورد، به‌صورت خودکار گزینه بعدی را امتحان می‌کند.

---

## شروع سریع

**گام ۱**: مدل خود را در IDE یا CLI روی `auto` تنظیم کنید:

```
model: "auto"
```

**گام ۲**: همین! بقیه کارها را RouteChi انجام می‌دهد.

**گام ۳** (اختیاری): برای وظایف خاص از یک variant استفاده کنید:

```
model: "auto/coding"    # بهترین برای کدنویسی
model: "auto/fast"      # سریع‌ترین پاسخ
model: "auto/cheap"     # ارزان‌ترین گزینه
```

---

## از کدام «auto» استفاده کنم؟

| اگر می‌خواهید... | از این استفاده کنید | بهترین برای | نحوه کار |
|----------------|-------------------|-----------|---------|
| **بهترین کلی** | `auto` | سؤالات عمومی، چت | سرعت، هزینه و کیفیت را متعادل می‌کند |
| **بهترین کد** | `auto/coding` | نوشتن کد، دیباگ | مدل‌هایی که در کدنویسی خوب هستند را انتخاب می‌کند |
| **سریع‌ترین پاسخ** | `auto/fast` | پاسخ‌های سریع، تأخیر کم | سرعت را به همه‌چیز ترجیح می‌دهد |
| **ارزان‌ترین گزینه** | `auto/cheap` | صرفه‌جویی در هزینه | ارزان‌ترین ارائه‌دهنده را انتخاب می‌کند |
| **با‌هوش‌ترین مدل** | `auto/smart` | وظایف پیچیده | کیفیت در اولویت + کشف مدل‌های جدید |
| **بیشترین در دسترس بودن** | `auto/offline` | وقتی ارائه‌دهندگان مشغول‌اند | ارائه‌دهندگانی با بیشترین ظرفیت را انتخاب می‌کند |

### مثال‌ها

```bash
# چت عمومی — متعادل
curl http://localhost:20128/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[{"role":"user","content":"Hello!"}]}'

# تولید کد — کیفیت در اولویت
curl http://localhost:20128/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"auto/coding","messages":[{"role":"user","content":"Write a Python function"}]}'

# پاسخ سریع — سرعت در اولویت
curl http://localhost:20128/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"auto/fast","messages":[{"role":"user","content":"What is 2+2?"}]}'
```

## نحوه کار (نسخه ساده)

وقتی درخواستی با `model: "auto"` ارسال می‌کنید، RouteChi این کارها را انجام می‌دهد:

1. **تمام ارائه‌دهندگان متصل شما را بررسی می‌کند** — هر ارائه‌دهنده‌ای که اضافه کرده‌اید (OpenAI، Anthropic، Google و غیره)
2. **هر کدام را بر اساس ۵ عامل امتیازدهی می‌کند**:
   - آیا کار می‌کند؟ (سلامت)
   - آیا ظرفیت دارد؟ (سهمیه)
   - چقدر هزینه دارد؟ (قیمت)
   - چقدر سریع است؟ (سرعت)
   - آیا در این وظیفه خوب است؟ (کیفیت)
3. **بهترین را انتخاب می‌کند** — ارائه‌دهنده‌ای که بالاترین امتیاز را گرفته، درخواست شما را دریافت می‌کند
4. **بازیابی خودکار** — اگر شکست بخورد، RouteChi به‌صورت خودکار گزینه بعدی را امتحان می‌کند

### سیستم امتیازدهی

هر ارائه‌دهنده امتیازی بین ۰ تا ۱ دریافت می‌کند. هرچه امتیاز بالاتر باشد، تناسب بهتر است.

| عامل | وزن | معنی |
|--------|--------|---------------|
| سلامت | ۲۰٪ | آیا ارائه‌دهنده کار می‌کند؟ (وضعیت circuit breaker) |
| سهمیه | ۱۵٪ | آیا ظرفیت باقی‌مانده دارد؟ |
| هزینه | ۱۵٪ | چقدر گران است؟ (ارزان‌تر = امتیاز بالاتر) |
| سرعت | ۱۲٪ | چقدر سریع است؟ (تأخیر کمتر = امتیاز بالاتر) |
| تناسب وظیفه | ۸٪ | آیا در این نوع وظیفه خوب است؟ |
| پایداری | ۵٪ | آیا یکنواخت است؟ (نرخ خطای پایین) |
| سطح | ۵٪ | سطح حساب (Ultra > Pro > Free) |
| سایر | ۲۰٪ | قرابت زمینه‌ای، تراکم اتصال و غیره |

### variantها چگونه امتیازدهی را تغییر می‌دهند

هر variant از وزن‌های متفاوتی استفاده می‌کند:

| Variant | اولویت با | وزن‌های کلیدی |
|---------|-------------|-------------|
| `auto` | متعادل | health=20%, quota=15%, cost=15% |
| `auto/coding` | کیفیت | taskFit=37%, stability=15% |
| `auto/fast` | سرعت | latency=32%, health=28% |
| `auto/cheap` | هزینه | cost=37% |
| `auto/smart` | کیفیت + کشف | taskFit=37%, exploration=10% |
| `auto/offline` | ظرفیت | quota=37%, health=28% |

## نحوه برخورد با شکست‌ها

RouteChi **سه لایه محافظتی** دارد:

### ۱. Fallback خودکار
اگر بهترین ارائه‌دهنده شکست بخورد، RouteChi به‌صورت خودکار گزینه بعدی را امتحان می‌کند. نیازی به کاری ندارید.

### ۲. خوددرمانی (Self-Healing)
اگر یک ارائه‌دهنده مدام شکست بخورد:
- **امتیاز < ۰.۲** → به مدت ۵ دقیقه مستثنی می‌شود
- **Circuit breaker باز است** → به‌صورت خودکار مستثنی می‌شود
- **بیش از ۵۰٪ ارائه‌دهندگان از کار افتاده‌اند** → حالت Incident (بدون اکتشاف)

### ۳. Fallback اضطراری
اگر همه ارائه‌دهندگان شکست بخورند، RouteChi به‌عنوان آخرین راه‌حل به ارائه‌دهندگان رایگان پایدار (مثل Kiro یا Qoder) هدایت می‌شود.

---

## پشتیبانی چند حسابی

اگر چندین حساب برای یک ارائه‌دهنده دارید (مثلاً دو کلید OpenAI)، RouteChi با هر کدام به‌عنوان **یک کاندیدای جداگانه** رفتار می‌کند. این یعنی:

- حساب A سهمیه دارد → از آن استفاده کن
- حساب B محدود شده → ردش کن
- حساب C ارزان‌تر است → ترجیحش بده

هر حساب به‌صورت مستقل بر اساس سلامت، سهمیه و سرعت خود امتیازدهی می‌شود.

---

## اکتشاف Bandit

RouteChi گاهی برای کشف گزینه‌های بهتر، ارائه‌دهندگان جدید را **کشف** می‌کند:

- **پیش‌فرض**: ۵٪ درخواست‌ها به ارائه‌دهندگان تصادفی می‌روند
- **Auto/smart**: نرخ اکتشاف ۱۰٪
- **غیرفعال** وقتی بیش از ۵۰٪ ارائه‌دهندگان ناسالم باشند

این به RouteChi کمک می‌کند تا یاد بگیرد کدام ارائه‌دهندگان برای الگوهای استفاده شما بهتر کار می‌کنند.

## سؤالات متداول

### «آیا همیشه گران‌ترین مدل را انتخاب می‌کند؟»

**خیر.** هزینه به‌صورت پیش‌فرض تنها ۱۵٪ امتیاز را تشکیل می‌دهد. یک ارائه‌دهنده ارزان، سریع و سالم می‌تواند یک ارائه‌دهنده گران را شکست دهد. اگر می‌خواهید هزینه را بیشتر در اولویت قرار دهید، از `auto/cheap` استفاده کنید.

### «اگر یک ارائه‌دهنده از کار بیفتد چه؟»

RouteChi به‌صورت خودکار آن را رد می‌کند و گزینه بعدی را امتحان می‌کند. اگر یک ارائه‌دهنده مدام شکست بخورد، موقتاً (۵ تا ۳۰ دقیقه) مستثنی می‌شود. نیازی به کاری ندارید.

### «آیا می‌توانم ببینم از کدام ارائه‌دهنده استفاده شده؟»

هدرهای پاسخ را بررسی کنید — RouteChi در هر پاسخ، ارائه‌دهنده و مدل استفاده‌شده را ذکر می‌کند.

### «آیا از استفاده من یاد می‌گیرد؟»

بله! سیستم امتیازدهی برای تصمیم‌گیری بهتر در طول زمان از داده‌های تاریخی (تأخیر، نرخ خطا، نرخ موفقیت) استفاده می‌کند.

### «تفاوت `auto` و `auto/smart` چیست؟»

- `auto` — متعادل، ۵٪ اکتشاف
- `auto/smart` — کیفیت در اولویت (همان وزن‌های `auto/coding`)، ۱۰٪ اکتشاف

وقتی بهترین کیفیت را می‌خواهید و با اکتشاف گاه‌به‌گاه مشکلی ندارید، از `auto/smart` استفاده کنید.

### «آیا می‌توانم یک ارائه‌دهنده خاص را اجبار کنم؟»

بله! به‌جای `auto` از یک کامبو با استراتژی `priority` استفاده کنید. برای جزئیات به [مرجع فنی](../routing/AUTO-COMBO.md) مراجعه کنید.

### «این چطور از round-robin متفاوت است؟»

Round-robin به‌ترتیب بین ارائه‌دهندگان چرخش می‌کند. کامبو خودکار **هر ارائه‌دهنده را امتیازدهی می‌کند** و بهترین را انتخاب می‌کند. هوشمندتر است — سلامت، سرعت، هزینه و کیفیت را در نظر می‌گیرد.
